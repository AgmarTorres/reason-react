Component as Prop

En ReactJS, <Menu banner=MyBanner /> es fácil; en ReasonReact, no podemos pasar trivialmente todo el módulo de un componente (ni siquiera sería sintácticamente válido). Un módulo reside en otra capa del lenguaje. Explicaciones). Solución:

let bannerCallback = (prop1, prop2) => <MyBanner message=prop1 count=prop2 />;

<Menu bannerFunc=bannerCallback />;

Esto conlleva algunas ventajas adicionales:

  • El propietario es el que controla cómo se renderiza el componente, no una opaca lógica dentro de MyBanner.
  • Evita la tendencia a necesitar usar expansión de props en el render de MyBanner, ya que pasar un componente a un hijo generalmente va acompañado de que ese hijo utilice expansión de props.
